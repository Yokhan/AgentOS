# Agent OS v0.3.15 - Live Execution Map v2

## Цель
Сделать живую карту исполнения основным операционным экраном: видно оркестратора, проектных агентов, возврат feedback, блокеры и реальные события без обрезания карты.

## Что исправляем
- Карта не должна показывать только строку оркестратора, если активные делегации уже ушли в архив.
- Горизонтальная шкала не должна быть фейковым `t+1`; события получают `ts_ms`, `offset_ms`, `time_label`, `event_index`.
- Runtime-текст из делегаций не должен приносить CP1251/UTF-8 mojibake в UI.
- Большая карта должна скроллиться внутри своего viewport, а не обрезаться родительскими контейнерами.

## Этапы
1. Backend: нормализовать display text, добавить поля времени в execution map v2.
2. Backend: включить `.delegation-archive.jsonl` в timeline, чтобы завершенные проектные ветки оставались видимыми.
3. Frontend: строить позиции по event index/time metadata, заменить `t+N` на реальные метки времени.
4. Frontend: дать stage-map собственный scroll viewport со sticky lane rail.
5. Verification: mojibake gate, UI smoke, Rust lib tests.

## Риски
- Архив делегаций может быть очень шумным, поэтому карта берет только свежий ограниченный срез.
- Старые клиенты должны пережить schema bump, поэтому `sequence` пока остается как compatibility/debug поле.
