# Agent OS v0.3.0: live route progress и операционный контроль

## Цель

Agent OS должен быть не статичным монитором, а рабочей панелью управления агентами: пользователь обсуждает задачу, выбирает исполнителя/ревьюера, запускает проектные маршруты и видит, что реально происходит сейчас.

Главный продуктовый результат `0.3.0`: каждый project-agent route должен показывать живую фазу работы, связь с делегацией, reviewer/cross-check, blockers, lease-состояние и следующий безопасный шаг.

## 20 крупных улучшений

1. Live route progress: lane показывает фазу `ready`, `queued`, `running`, `verifying`, `reviewing`, `needs_user`, `blocked`, `done`, `idle`.
2. Единая семантика маршрута: route state больше не сводится только к счетчикам work items.
3. Связь route -> work item -> delegation: UI показывает активную делегацию и следующий work item.
4. Явное `needs_user`: permission, failed, rejected и cancelled не прячутся в generic blocked.
5. Cross-check lifecycle: executor finished и reviewer verdict видны как отдельные стадии.
6. Reviewer verdict visibility: approve/warn/fail становится частью route progress.
7. Lease visibility: active write leases влияют на фазу и риск маршрута.
8. Route progress aggregate: orchestration map получает сводку по live-фазам.
9. Management leverage учитывает live state, а не только статические счетчики.
10. Control load становится практичнее: user attention растет от `needs_user`, а не от любого шума.
11. UI показывает компактную дорожку выполнения вместо очередного cockpit-блока.
12. Route click становится понятнее: queue next, monitor, resolve user blocker, review verdict.
13. Prompt fallback становится русскоязычным и operational, а не абстрактным.
14. Project-agent routes визуально разделяют готовую, активную, проверяемую и заблокированную работу.
15. Сводка `live progress` показывает, где система сейчас реально занята.
16. Timeline и orchestration map получают общий big-plan этап `live_route_progress`.
17. Backend tests фиксируют фазовую модель, чтобы UX не разъехался после следующих релизов.
18. UI tests/static checks защищают chat bundle от синтаксических регрессий.
19. Release notes объясняют `0.3.0` как продуктовый milestone, а не мелкий patch.
20. Версия `0.3.0` выпускается только после полного прохода и updater build.

## Acceptance criteria

- Route lane отвечает на вопрос: что сейчас происходит, кто делает, кто проверяет, где нужен пользователь.
- `needs_user` виден отдельно и не теряется среди generic blockers.
- Management card учитывает live progress и рекомендует следующий операционный шаг.
- Chat UI показывает движение маршрутов компактно, без новой перегруженной кабины.
- Релиз `v0.3.0` выпущен только после тестов и updater build.
