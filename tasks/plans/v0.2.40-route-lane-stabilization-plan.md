# Agent OS v0.2.40 Route Lane Stabilization Plan

## Goal

Make project-agent routing feel like an executable control surface, not a static diagnostic card. A lane click must either queue real work immediately or explain why the lane only supports a prompt fallback.

## 20 Top-Level Improvements

1. Make route lanes executable from chat without hidden submenus.
2. Keep solo mode provider-neutral: any selected provider can orchestrate if it has write access.
3. Treat chat as the universal command surface, not a mode-specific tool panel.
4. Show all relevant project-agent lanes, with a compact default and explicit expansion.
5. Preserve blocked delegations even when no session/work-item lane exists.
6. Separate "running" state from "has blockers" so active work is not hidden by old failures.
7. Put the next executable work item on every lane when possible.
8. Make fallback behavior visible when no executable work item exists.
9. Refresh orchestration map and execution timeline after lane actions.
10. Surface route counts as operational signals: tasks, leases, blockers.
11. Make route cards scan-friendly under heavy project load.
12. Make code-context availability visible next to route decisions.
13. Keep planned work persisted in repo files, not only in chat.
14. Add backend tests for route-state semantics and blocker-only lanes.
15. Add frontend syntax checks for new chat imports/actions.
16. Keep release notes tied to user-visible behavior, not internal churn.
17. Avoid another cockpit: one lane, one primary action, details on demand.
18. Preserve existing graph/context/plan flows while making routes actionable.
19. Prepare the route model for future DeepSeek/extra providers without hardcoding Codex.
20. Ship through the updater pipeline with version bump, tag, release, and CI verification.

## Exit Criteria

- Clicking a route lane with a ready work item queues it directly.
- Blocked delegations are visible even without a normal lane.
- Running lanes remain running while still exposing blockers.
- The UI can show all returned routes without wasting default space.
- Tests and updater build pass before release.
