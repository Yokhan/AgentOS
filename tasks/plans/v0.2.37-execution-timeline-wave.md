# v0.2.37 Execution Timeline Wave

Date: 2026-04-26
Owner: AgentOS
Release target: v0.2.37

## Goal

Make active work readable as a normalized execution timeline instead of scattered
chat text, stream bubbles, Duo events, and delegation state. This is stage 4/6
of the big UX/orchestration plan.

## Current Big-Plan Position

- Completed: foundation, Route Card, live transcript, orchestration map.
- Current: execution timeline + event normalization.
- Next: backend event unification and provider/ACP adapter expansion.

## Top-Level Improvements

1. Add a read-only backend execution timeline command.
2. Normalize chat stream events into user-readable timeline rows.
3. Include Duo/session events when a room session is active.
4. Include recent delegation lifecycle events for the current route.
5. Include delegation stream stage/done events where available.
6. Expose stage 4/6 in backend big-plan metadata.
7. Add frontend timeline state and refresh loop.
8. Render a compact Execution Timeline Card in chat.
9. Highlight blockers/warnings/cancelled/done states.
10. Keep this wave read-only: no new execution source of truth.

5. Read relevant delegations from state.
6. Read recent `tasks/.stream-deleg-{id}.jsonl` stage/done events.

8. Register command in Tauri invoke handler.
9. Add backend tests for stream normalization.
10. Add `executionTimeline` signal.
11. Add `loadExecutionTimeline()` API helper.
12. Refresh timeline during active runs, Duo busy states, project switches.
13. Add `ExecutionTimelineCard` below orchestration map.
14. Show big-plan stage 4/6 and item count.
15. Add "refresh timeline" and "copy summary" controls.
16. Add CSS for timeline rows and status severity.
17. Bump version to `0.2.37`.
18. Update release notes.

20. Commit, tag, push, verify release assets and `latest.json`.

## Acceptance Criteria

- A user can see what an agent is doing without reading raw stream dumps.
- A user can distinguish chat run events, Duo room events, and delegation events.
- Blockers/warnings/cancelled/done states are visible at a glance.
- The product itself says we are at big-plan stage 4/6.
- `v0.2.37` updater assets publish successfully.
