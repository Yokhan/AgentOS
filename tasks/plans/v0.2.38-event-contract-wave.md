# v0.2.38 Event Contract Wave

Date: 2026-04-26
Owner: AgentOS
Release target: v0.2.38

## Goal

Move from a timeline-specific parser to a shared event contract that chat,
Duo rooms, delegation streams, and future project agents can all use. This is
stage 5/6 of the big UX/orchestration plan.

## Current Big-Plan Position

- Completed: foundation, Route Card, live transcript, orchestration map,
  execution timeline.
- Current: event contract + normalized source adapters.
- Next: project-agent routing and provider adapter expansion.

## Top-Level Improvements

1. Add a shared backend event contract module.
2. Define one normalized timeline/event row shape.
3. Normalize chat stream events through that module.
4. Normalize Duo session events through that module.
5. Normalize delegation state through that module.
6. Normalize delegation stream stage/done/safety/tool events through that module.
7. Preserve raw events only as source input, not UI contract.
8. Add schema/status metadata so UI can explain what is covered.
9. Move visible big-plan stage to 5/6.
10. Keep the wave read-only and backwards compatible with existing JSONL files.

- Timeline UI is driven by one shared backend contract, not ad-hoc parsing.

- The product itself says we are at big-plan stage 5/6.
