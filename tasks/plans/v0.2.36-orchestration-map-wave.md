# v0.2.36 Orchestration Map Wave

Date: 2026-04-26
Owner: AgentOS
Release target: v0.2.36

## Goal

Make the chat explain how work is routed across projects, plans, work items,
delegations, Duo sessions, and graph/code context. The user should see the
current layer of the big plan directly in the interface and in progress reports.

## Current Big-Plan Position

- Completed: chat route card, live transcript, safer stop/output handling.
- Current: project routing + plan/work-item visibility + code context status.
- Next: richer execution timeline and backend event unification.

## Top-Level Improvements

1. Add a read-only orchestration map backend command.
2. Connect the map to UI state and refresh it with scope/session changes.
3. Show plan progress as a visible staged tracker.
4. Show project session and work-item counts for the active route.
5. Show live/pending/failed delegation counts in the same route context.
6. Show graph/code-context readiness instead of hiding it behind `[GRAPH_CONTEXT]`.
7. Make the UI explain which project agent/session a task would enter.
8. Provide context actions: attach graph context, open project graph, verify graph.
9. Make big-plan stage visible in feedback instead of only in release notes.
10. Keep the implementation read-only for the map to avoid creating another source of truth.

## Concrete Changes

1. Add `get_orchestration_map` Tauri command.
2. Include active scope in the map by reusing `resolve_active_scope`.
3. Include active plan counts and next pending/running step.
4. Include project sessions relevant to current project/session.
5. Include work items relevant to current project/session.
6. Include delegation counts and recent active delegations.
7. Include file lease counts for execution safety.
8. Include graph/code-context status for current project.
9. Register the command in Tauri invoke handler.
10. Add `orchestrationMap` signal to frontend store.
11. Add `loadOrchestrationMap()` API helper.
12. Refresh map on project/session/scope changes and after Duo actions.
13. Add `OrchestrationMapCard` in chat below transcript route surfaces.
14. Show big-plan stage: Foundation done, Routing current, Timeline next.
15. Show project/session/work-item/delegation/graph rows.
16. Add route-aware buttons: attach code context, open graph, verify graph, open plans.
17. Add warnings when graph context is missing or work is blocked by active leases.
18. Add release notes and version bump.
19. Add tests for backend map command where feasible.
20. Run JS checks, Rust tests, updater build, tag, release.

## Acceptance Criteria

- The user can tell whether AgentOS is operating globally, inside a project,
  inside a plan, or inside a work item.
- The UI shows whether graph/code context exists for the current project.
- The UI shows which work is queued/running/blocked before asking agents to act.
- The big-plan stage is visible in the product and in assistant progress reports.
- Release assets and updater manifest publish for `v0.2.36`.
